
## Блокировки РКН на разных уровнях OSI

| level | blocks                             |
| ----- | ---------------------------------- |
| l1    | -                                  |
| l2    | -                                  |
| l3    | блокировки по ip адресам           |
| l4    | блокировки протоколов, vpn         |
| l5    | dpi, анализирующий TLS соединения  |
| l6    | dpi                                |
| l7    | DPI по имени хоста, DNS блокировки |

## Модель OSI
### Физический уровень (L1)

Передаёт биты данных по физической среде (кабели, радиоволны). Управляет электрическими, механическими и оптическими характеристиками сигнала, без обнаружения ошибок.[](https://net.e-publish.ru/p217aa1.html)​

### Канальный уровень (L2)

Организует передачу кадров между соседними узлами, добавляя MAC-адреса. Обеспечивает обнаружение и исправление ошибок, контроль потока.[](https://timeweb.com/ru/community/articles/setevye-modeli-osi)​

### Сетевой уровень (L3)

Определяет маршруты пакетов между сетями с использованием IP-адресов. Решает задачи логической адресации и фрагментации данных.[](https://www.oslogic.ru/knowledge/809/setevoj-uroven-modeli-osi-obshhie-ponyatiya/)​

### Транспортный уровень (L4)

Гарантирует надёжную доставку данных (TCP/UDP), управляет соединениями, сегментацией и повторной передачей.[](https://timeweb.com/ru/community/articles/setevye-modeli-osi)​

### Сеансовый уровень (L5)

Управляет сеансами связи: установкой, поддержкой и завершением. Синхронизирует данные и восстанавливает соединения при разрывах.[](https://altaev-aa.narod.ru/net/lecture2.pdf)​

### Уровень представления (L6)

Преобразует данные для приложений: шифрование, сжатие, кодирование (ASCII/Unicode). Обеспечивает совместимость форматов.[](https://timeweb.com/ru/community/articles/setevye-modeli-osi)​

### Прикладной уровень (L7)

Предоставляет сервисы пользователям (HTTP, FTP, SMTP). Интерферирует приложения с сетью.
